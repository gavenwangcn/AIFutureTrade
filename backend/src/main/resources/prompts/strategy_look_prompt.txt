### 6.1）K 线形态与交叉类判断

以下逻辑**仅用于判断是否发出通知**，不表示开多/开空/平仓；满足 `strategy_context` 时在 `decisions` 中填 `signal: "notify"`，并在 `justification` / `key_date` 中写明**形态名称与数值**。

#### 通用前提
- **至少 2 根 K 线**：需同时取 **当前 K**（一般为 `klines[-1]`）与 **前一根 K**（`klines[-2]`），交叉类判断才有意义。
- **所有参与比较的指标**在两根 K 上都必须 **非 `None`**，否则打 `warning` 并跳过该形态判定。
- **⚠️⚠️⚠️ 交叉类比较必须用「含等于」的不等式（与 buy/sell 一致）**  
  金叉/死叉/上穿/下穿的**当前根与前一根**两侧比较，**一律使用 `>=` 与 `<=`**，**禁止**单独使用严格的 `>` 或 `<`（否则会漏掉**相切**即两线数值相等的交叉）。  
  - ❌ 错：`ema20_prev > ema30_prev`（前一根漏了等于）  
  - ✅ 对：`ema20_prev >= ema30_prev` 与 `ema20_current <= ema30_current` 等组合，且语义与「上穿/下穿」一致。

#### MA（或 EMA）金叉 / 死叉
- **金叉（快线上穿慢线）**：当前根 **快线 ≥ 慢线** 且 前一根 **快线 ≤ 慢线**。  
  典型参数：`ma5` vs `ma20`，或 `ema5` vs `ema20`（以 `strategy_context` 为准）。
```python
cur, prev = klines[-1], klines[-2]
ma_f_cur = cur.get("indicators", {}).get("ma", {}).get("ma5")
ma_s_cur = cur.get("indicators", {}).get("ma", {}).get("ma20")
ma_f_prev = prev.get("indicators", {}).get("ma", {}).get("ma5")
ma_s_prev = prev.get("indicators", {}).get("ma", {}).get("ma20")
if None in (ma_f_cur, ma_s_cur, ma_f_prev, ma_s_prev):
    self.log.warning(f"{symbol} MA 值不足，跳过金叉死叉判定")
else:
    is_ma_golden = ma_f_cur >= ma_s_cur and ma_f_prev <= ma_s_prev
    is_ma_death = ma_f_cur <= ma_s_cur and ma_f_prev >= ma_s_prev
    self.log.info(f"{symbol} MA交叉: 金叉={is_ma_golden}, 死叉={is_ma_death}, 快={ma_f_cur},慢={ma_s_cur}")
```
- **EMA** 写法相同，把 `ma` 换成 `ema`，字段如 `ema5`/`ema20`/`ema30` 等。

#### MACD：DIF 与 DEA 金叉 / 死叉
- **金叉**：`dif_current >= dea_current and dif_prev <= dea_prev`（DIF 上穿 DEA）。  
- **死叉**：`dif_current <= dea_current and dif_prev >= dea_prev`（DIF 下穿 DEA）。  
- 从 `indicators.macd` 读取 `dif`、`dea`、`bar`；**均需 None 检查**。若 K 线不足，MACD 可能全为 `None`（参见 buy 文档说明）。

```python
def _macd_pair(kline):
    m = kline.get("indicators", {}).get("macd") or {}
    return m.get("dif"), m.get("dea")

dif_c, dea_c = _macd_pair(cur)
dif_p, dea_p = _macd_pair(prev)
if None in (dif_c, dea_c, dif_p, dea_p):
    self.log.warning(f"{symbol} MACD 不全，跳过 MACD 交叉判定")
else:
    macd_golden = dif_c >= dea_c and dif_p <= dea_p
    macd_death = dif_c <= dea_c and dif_p >= dea_p
    self.log.info(f"{symbol} MACD: 金叉={macd_golden}, 死叉={macd_death}")
```

#### KDJ：K 与 D 金叉 / 死叉
- **金叉（K 上穿 D）**：`k_cur >= d_cur and k_prev <= d_prev`（**含等于**，与 buy 中规则3 一致）。  
- **死叉（K 下穿 D）**：`k_cur <= d_cur and k_prev >= d_prev`。

```python
def _kdj(kline):
    kdj = kline.get("indicators", {}).get("kdj") or {}
    return kdj.get("k"), kdj.get("d")

k_c, d_c = _kdj(cur)
k_p, d_p = _kdj(prev)
if None in (k_c, d_c, k_p, d_p):
    self.log.warning(f"{symbol} KDJ 不全，跳过 KDJ 交叉判定")
else:
    kdj_golden = k_c >= d_c and k_p <= d_p
    kdj_death = k_c <= d_c and k_p >= d_p
    self.log.info(f"{symbol} KDJ: K上穿D={kdj_golden}, K下穿D={kdj_death}, K={k_c:.4f}, D={d_c:.4f}")
```

#### +DI / −DI 交叉（趋势辅助，可选）
- 从 `indicators.adx` 读取 `+di14`、`-di14`（字段名与 buy 一致）。  
- **+DI 上穿 −DI**：`plus_c >= minus_c and plus_p <= minus_p`（两侧含等于）。  
- **+DI 下穿 −DI**：`plus_c <= minus_c and plus_p >= minus_p`。  
- 需与 `adx14` 等配合时，在 `strategy_context` 中约定阈值，并打日志。

#### 多周期共振（示例思路）
- 若要求「例如 1h 死叉且 15m 同向」，应对 **各周期分别取 `klines`**，分别算交叉布尔值，再 `and`，且 **每个周期单独打日志**，便于企微与复盘。

#### 写入 `notify` 时的建议
- `key_date` 中保存：`{"pattern": "MA金叉", "timeframe": "1h", "fast": ma_f_cur, "slow": ma_s_cur, ...}`  
- `justification` 用**中文简述**触发的形态与关键数值（如「1h MA5 上穿 MA20，MA5=…, MA20=…」）。

### 7）多周期联合示例（指导性伪代码）
```python
def execute_look_decision(self, symbol: str, market_state: Dict) -> Dict[str, List[Dict]]:
    decisions = {}
    sym = (symbol or "").strip().upper()
    st = market_state.get(sym) or market_state.get(symbol)
    if not isinstance(st, dict):
        self.log.warning(f"market_state 无 {sym} 条目")
        return decisions

    price = st.get("price")
    contract = st.get("contract_symbol") or f"{sym}USDT"
```
